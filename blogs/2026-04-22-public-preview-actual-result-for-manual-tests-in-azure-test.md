<p>We&#8217;re excited to announce the public preview of the highly anticipated Actual Result (AR) feature for manual testing in Azure Test Plans! This feature has been one of the <a href="https://developercommunity.visualstudio.com/t/actual-result-field-needed-in-test-results-test-pl/890708" target="_blank">top requests</a> of the community, and we&#8217;re thrilled to make it available for you.</p>
<h3>Why use the Actual Result feature?</h3>
<p>Manual testing is a critical part of many teams&#8217; processes. The new feature via the Actual Result field enables you to record precise outcomes for each test step, improving traceability, audit readiness, and collaboration across your teams.</p>
<p><a href="https://devblogs.microsoft.com/devops/wp-content/uploads/sites/6/2026/04/Screenshot-2026-04-10-at-14.31.25.webp"><img alt="Screenshot 2026 04 10 at 14 31 25 image" class="alignnone size-medium wp-image-72643" height="452" src="https://devblogs.microsoft.com/devops/wp-content/uploads/sites/6/2026/04/Screenshot-2026-04-10-at-14.31.25-300x226.webp" width="600" /></a></p>
<h3>Key Functionalities</h3>
<ul>
<li><strong>Step-Level Actual Results</strong>: Capture exactly what happened during manual execution for each test step, via text and attachments.</li>
<li><strong>Configurable at Test Plan Level</strong>: Enable AR as an optional or mandatory field for all test cases under a specific test plan.</li>
<li><strong>Audit and Compliance Support</strong>: Retain actual results alongside test run data, supporting traceability and compliance needs.</li>
<li><strong>Easy Review</strong>: Review actual results in the Test Run Hub&#8217;s test case results pages, with clear indicators of AR status.</li>
<li><strong>Programmatic Access</strong>: Fetch actual results via the Azure DevOps REST API for automation and integration scenarios.</li>
</ul>
<h3>Getting Started</h3>
<p>To try out the Actual Result field:</p>
<ul>
<li>Open your test plan in Azure Test Plans.</li>
<li>Go to Test Plan Settings (Azure Test Plan License Needed) and choose your preferred AR mode.</li>
<li>Start a manual test run and experience the new AR field in action.</li>
<li>Review actual results after test completion in the Test Run Hub.</li>
</ul>
<p>For all the details, check the public <a href="https://go.microsoft.com/fwlink/?linkid=2352773">documentation</a> page of the feature.</p>

<h3>What Does Public Preview Mean?</h3>
<p>This feature is in Public Preview. It means that functionality may change or be discontinued without notice. Preview features are provided without a Service Level Agreement and with limited support.</p>
<p>Additionally, if the feature is not yet available in your Azure DevOps organization, please allow a few more days, as it is being rolled out gradually.</p>
